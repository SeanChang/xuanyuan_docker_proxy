# 抖音直播循环值守录制工具  


## 功能概述  
这是一款针对抖音直播的录制工具（项目地址：[[]] ），核心能力是实现“循环值守”——即自动监控指定直播间状态，一旦主播开播便自动启动录制，直播结束后持续后台监控，下次开播时无需手动操作即可再次录制。适合需要长期跟踪某主播直播内容、但不方便手动值守的场景（比如错过直播时段仍需留存回放）。  


## 使用步骤  
### 1. 获取工具源码  
直接从 GitHub 仓库克隆或下载项目源码到本地（仓库提供完整代码和说明文档）。  

### 2. 配置运行环境  
- 确保本地已安装 Python（推荐 3.8+ 版本）及相关依赖库（项目 `requirements.txt` 文件中列出了所需依赖，通过 `pip install -r requirements.txt` 一键安装即可）。  

### 3. 设置直播间信息  
打开项目中的配置文件（通常是 `config.py` 或类似命名的文件），按说明填写需要监控的直播间 ID（可从抖音直播页面 URL 中获取，例如 `[] 中的 `123456` 即为直播间 ID）。  

### 4. 启动值守录制  
运行项目主程序（通常是 `main.py`），工具会自动开始监控直播间状态：  
- 若直播间当前在线，立即启动录制，视频文件默认保存在本地指定目录（可在配置文件中修改保存路径）；  
- 若直播间未开播，工具将进入循环等待状态（等待间隔可配置），持续检测直播状态，直到主播开播后自动触发录制。  


## 注意事项  
- **环境依赖**：务必按项目要求安装完整依赖，避免因缺少库导致录制失败；  
- **直播间 ID 准确性**：配置时需确认直播间 ID 正确，错误 ID 会导致监控失效；  
- **网络权限**：确保工具能正常访问抖音直播接口（部分网络环境可能需要配置代理）；  
- **存储空间**：长期录制需预留足够本地存储空间，避免因磁盘满导致录制中断。  


通过以上步骤，即可实现对指定抖音直播间的无人值守录制，无需手动干预开播/停播环节。
