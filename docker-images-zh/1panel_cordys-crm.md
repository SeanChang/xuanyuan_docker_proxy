# Cordys CRM 技术文档


## 镜像概述与主要用途

Cordys CRM 是由飞致云匠心出品的新一代开源 AI CRM 系统，集信息化、数字化、智能化于一体，定位为 Salesforce CRM 的开源替代方案。其名称由“Cord”（连接之绳）与“System”（系统）融合而成，寓意“关系的纽带系统”，核心目标是帮助企业连接客户、缔造长期价值。

> 注：Cordys CRM 目前处于公测阶段，预计于 2025 年底正式开源并发布。


## 核心功能与特性

### 灵活易用
- **现代化体验**：提供直观的用户界面与流畅的操作流程，降低学习成本。
- **高度可配置**：支持自定义表单、业务流程与权限体系，无需代码即可适配企业个性化需求，快速实现销售自动化。

### 安全可控
- **私有化部署**：所有客户数据与业务信息存储于企业自有服务器，杜绝数据外泄风险。
- **完全数据主权**：企业对数据拥有完全控制权，满足合规性要求与数据安全需求。

### BI 数据分析能力
- **嵌入式 BI 工具**：集成 DataEase 强大的嵌入式分析能力，支持多维度数据可视化。
- **智能问数**：结合 SQLBot 实现自然语言查询数据，无需编写 SQL，提升数据分析效率。

### AI 智能体支持
- **开放 MCP Server**：提供标准化接口，支持集成外部 AI 能力。
- **MaxKB 集成**：借助 MaxKB 智能体开发平台，快速构建销售助手、客户服务机器人等各类智能体，赋能销售全流程。


## 使用场景与适用范围

### 适用场景
- **中小企业客户关系管理**：低成本实现客户信息统一管理、跟进记录追踪、销售流程规范化。
- **销售团队自动化**：通过可配置流程自动化销售任务分配、商机推进提醒、合同管理等。
- **企业数据分析决策**：利用嵌入式 BI 工具实时监控销售业绩、客户转化率等关键指标，辅助管理层决策。
- **智能销售辅助**：通过 AI 智能体实现客户需求分析、产品推荐、邮件自动生成等，提升销售效率。

### 适用范围
- 需要私有化部署、保障数据安全的企业。
- 希望替代 Salesforce 等商业 CRM、降低软件采购成本的组织。
- 对销售数据分析有强需求的团队。
- 希望引入 AI 技术提升销售效能的企业。


## 使用方法与配置说明

### 安装部署

#### 环境要求
- 操作系统：Linux 服务器（推荐 CentOS 7+/Ubuntu 18.04+）。
- 依赖软件：Docker（需提前安装，参考 [Docker 官方安装文档](https://docs.docker.com/get-docker/)）。
- 硬件配置：建议至少 4 核 CPU、8GB 内存、50GB 可用磁盘空间。


#### Docker 命令行部署
通过以下命令一键部署 Cordys CRM 容器：

```bash
docker run -d \
  --name cordys-crm \
  --restart unless-stopped \  # 容器退出时自动重启（除非手动停止）
  -p 8081:8081 \              # 映射应用 Web 端口（主机端口:容器端口）
  -p 8082:8082 \              # 映射内部服务通信端口
  -v ~/cordys:/opt/cordys \   # 挂载数据卷（持久化存储配置、日志、业务数据等）
  1panel/cordys-crm           # 镜像名称
```

**参数说明**：
- `--name cordys-crm`：指定容器名称为 `cordys-crm`。
- `--restart unless-stopped`：设置容器重启策略，除非手动停止，否则退出后自动重启。
- `-p 8081:8081`：将容器内 8081 端口映射到主机 8081 端口，用于 Web 访问。
- `-p 8082:8082`：内部服务通信端口，确保容器间或外部集成时的服务可用性。
- `-v ~/cordys:/opt/cordys`：将主机 `~/cordys` 目录挂载到容器内 `/opt/cordys`，用于持久化存储数据（避免容器删除后数据丢失）。


#### 1Panel 应用商店部署
通过 1Panel 服务器运维平台的应用商店快速安装：
1. 登录 1Panel 控制台，进入「应用商店」。
2. 搜索「Cordys CRM」，点击「安装」并按照指引完成配置。


#### 离线环境部署
在无网络或受限网络环境中，可通过离线安装包部署：
1. 下载离线安装包：访问 [Cordys CRM 社区页面](https://community.fit2cloud.com/#/products/cordys-crm/information) 获取最新离线包。
2. 参考离线包内的 `README.md` 文档执行部署步骤。


### 访问系统
部署完成后，通过以下方式访问 Cordys CRM：

1. **Web 访问**：在浏览器中输入 `http://<服务器 IP>:8081/`（替换 `<服务器 IP>` 为实际主机 IP）。
2. **默认账号**：
   - 用户名：`admin`
   - 密码：`CordysCRM`（首次登录后建议立即修改）。


### 配置参数说明
当前版本通过 Docker 命令行参数主要配置端口映射与数据卷挂载，暂无额外环境变量需手动配置。如需自定义高级配置（如数据库连接、外部服务集成），可通过容器内 `/opt/cordys/conf` 目录下的配置文件调整（需重启容器生效）。


## 注意事项
- 数据备份：通过数据卷 `~/cordys` 定期备份数据，避免数据丢失。
- 端口冲突：若主机 8081/8082 端口已被占用，可修改 `-p` 参数中的主机端口（如 `-p 8083:8081` 映射至 8083 端口）。
- 安全建议：生产环境中建议通过反向代理（如 Nginx）配置 HTTPS，增强访问安全性。
