# Refly.AI Web 前端

Refly.AI Web 是 Refly.AI 平台的前端 Web 界面组件，提供直观的可视化画布和用户交互界面，让非技术创作者能够轻松构建、管理和分享 AI 自动化工作流。

## 核心功能与特性

### 可视化画布界面

**直观的可视化编辑器**：提供类似 Canva 的拖拽式界面，让用户通过简单的拖放操作构建复杂的工作流，无需编写任何代码。

**实时可视化执行**：工作流的每一步执行过程都在画布上清晰可见，支持实时查看工作流状态、执行进度和结果反馈。

**画布交互操作**：支持缩放、平移、多选等丰富的画布操作，提供流畅的用户体验。

### 工作流构建与管理

**智能节点连接**：通过可视化连线方式连接不同的智能体节点，构建完整的工作流逻辑。

**工作流副驾驶界面**：集成工作流副驾驶功能，用户可以通过自然语言描述任务，系统自动在画布上生成对应的工作流。

**工作流模板与市场**：提供工作流模板库和市场界面，用户可以浏览、使用和分享工作流模板。

### 实时干预与控制

**执行控制面板**：提供暂停、恢复、重启等执行控制功能，支持在工作流执行过程中进行实时干预。

**状态监控**：实时显示每个节点的执行状态、输入输出数据，便于调试和优化工作流。

**历史记录查看**：支持查看工作流执行历史，包括执行日志、错误信息和性能数据。

### 用户界面特性

**响应式设计**：适配桌面、平板和移动设备，提供一致的用户体验。

**多主题支持**：支持明暗主题切换，满足不同用户的视觉偏好。

**国际化支持**：提供多语言界面，支持全球用户使用。

## 技术架构

Refly.AI Web 作为前端组件，与 `reflyai/refly-api` 后端服务配合使用，共同构成完整的 Refly.AI 平台。

- **前端框架**：基于现代 Web 技术栈构建，提供流畅的交互体验
- **API 集成**：与 Refly API 服务无缝集成，实现前后端数据交互
- **实时通信**：支持 WebSocket 等实时通信协议，实现工作流状态的实时更新

## 部署与使用

### 与 API 服务配合使用

Refly Web 需要与 `reflyai/refly-api` 服务配合部署，典型的部署架构包括：

```yaml
services:
  refly-api:
    image: reflyai/refly-api:latest
    # API 服务配置
    
  refly-web:
    image: reflyai/refly-web:latest
    ports:
      - "3000:3000"
    environment:
      - API_URL=http://refly-api:8000
    depends_on:
      - refly-api
```

### 环境变量配置

- `API_URL`：Refly API 服务的地址，用于前后端通信
- `PUBLIC_URL`：Web 应用的公共访问地址
- `ENABLE_ANALYTICS`：是否启用分析功能（可选）

### 访问 Web 界面

部署完成后，通过浏览器访问 Web 界面（默认端口 3000），即可开始使用可视化画布构建工作流。

## 功能亮点

- **零代码工作流构建**：通过可视化界面，无需编写代码即可创建复杂的工作流
- **所见即所得**：所有操作和结果都在画布上实时显示，提供直观的反馈
- **协作功能**：支持多人协作编辑工作流，实时同步更改
- **工作流分享**：一键将工作流发布到市场，与其他用户分享
- **模板系统**：丰富的模板库，快速启动常见工作流场景

## 与 Refly API 的关系

Refly Web 是 Refly.AI 平台的前端界面，负责用户交互和可视化展示；Refly API 是后端服务，负责工作流的执行和数据处理。两者配合使用，共同提供完整的 Vibe 工作流平台体验。
