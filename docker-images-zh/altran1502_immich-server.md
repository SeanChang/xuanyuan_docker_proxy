# Nest.js Immich服务器：自托管移动资产备份与展示方案


## 简介  
Immich 是一款基于 Nest.js 框架开发的自托管工具，专门用来解决手机照片、视频等移动资产的备份和管理问题。它的核心优势在于“自托管”——所有文件都存在你自己的服务器上，不用依赖第三方云服务，数据隐私和控制权完全在自己手里。Nest.js 的稳定架构也让它能高效处理大量媒体文件的存储、同步和展示。


## 核心功能  
简单说，它主要帮你解决这些实际问题：  

### 1. 手机资产自动备份  
手机里的照片、视频，打开 Immich 客户端后，会按你设定的规则（比如仅 Wi-Fi 环境、充电时）自动上传到服务器，不用手动拷贝，避免误删或手机丢失导致文件丢失。  

### 2. 多设备文件统一管理  
不管是安卓手机、苹果手机，还是平板，登录同一个 Immich 账号后，所有设备的媒体文件会汇总到一个库里。换手机时不用手动导照片，新手机登录后直接同步历史文件。  

### 3. 本地存储，空间自己说了算  
所有文件都存在你指定的服务器硬盘路径下，没有云存储的空间限制（只要硬盘够大）。支持外接硬盘扩展存储，成本比买云空间低很多。  

### 4. 相册展示与快速检索  
上传后的文件会按拍摄时间、地点自动分类，Web 界面和手机客户端都能直接浏览。支持按关键词搜索（比如“海边”“生日”），也能手动创建相册、分享给家人（对方不用注册，用链接就能访问）。  


## 部署前要准备什么  
自托管需要一点基础设备和环境，提前准备好这些，部署会更顺利：  

### 硬件要求  
- **服务器**：普通电脑、NAS（网络存储服务器）、树莓派都能用。最低配置建议 2 核 CPU + 4G 内存（如果备份文件多，内存建议 8G 以上，避免卡顿）。  
- **存储**：至少预留 100G 空间（按你手机现有照片视频总量估算，建议用 SSD 或高速 HDD，读写速度快）。  
- **网络**：服务器需要联网，手机和服务器在同一局域网内备份速度更快；如果想在外网（比如公司）访问，需要在路由器做端口映射（或用内网穿透工具）。  

### 软件环境  
- 安装 Docker 和 Docker Compose（推荐用 Docker 部署，不用手动配置 Node.js、数据库等依赖，官网有详细安装教程，跟着一步步来就行）。  
- 服务器系统：Linux（如 Ubuntu、Debian）最稳定，Windows 或 macOS 也能用，但 Linux 对 Docker 的支持更好。  


## 具体部署步骤  
按以下步骤操作，新手也能搞定：  

### 1. 下载配置文件  
先在服务器上建一个存放 Immich 配置的文件夹（比如 `/opt/immich`），然后从 Immich 官网或 GitHub 仓库下载两个核心文件：`docker-compose.yml`（部署脚本）和 `.env`（环境配置文件）。可以用 `wget` 命令直接下载，也可以在浏览器下载后传到服务器。  

### 2. 修改配置参数  
打开 `.env` 文件，重点改这几个关键参数（用记事本或 VS Code 编辑就行）：  
- `UPLOAD_LOCATION`：设置媒体文件的存储路径，比如 `/mnt/hdd/immich-files`（确保这个路径有读写权限，比如用 `chmod 777 /mnt/hdd/immich-files` 授权）。  
- `PORT`：Web 管理界面的访问端口（默认 3001，若被占用可改其他端口，比如 3002）。  
- `DB_PASSWORD`：数据库密码，随便设一个复杂点的（比如 `ImmichDB@2024`），避免被破解。  

### 3. 启动服务  
在配置文件所在的文件夹（比如 `/opt/immich`）打开终端，执行命令 `docker-compose up -d`。Docker 会自动下载所需的镜像（第一次可能慢点，耐心等），然后启动服务。启动后用 `docker-compose ps` 命令检查，看到所有容器状态是 `healthy` 就说明启动成功了。  

### 4. 初始化设置  
在浏览器里输入 `服务器IP:端口`（比如 `192.168.1.100:3001`），访问 Immich 的 Web 界面。首次打开会提示注册管理员账号，按提示填完信息后，下载手机客户端（安卓在官网或 Google Play，苹果在 App Store），用管理员账号登录。最后在客户端里开启“自动备份”，选要备份的相册，设置触发条件（比如“仅 Wi-Fi 时上传”“充电时优先备份”）。  


## 使用小技巧  
部署完后，这些实用操作能让体验更好：  

- **定期备份服务器数据**：虽然 Immich 是备份工具，但服务器硬盘坏了会丢文件。建议用 NAS 的 RAID 功能（硬盘冗余），或定期把 `UPLOAD_LOCATION` 路径下的文件拷到另一个硬盘/云盘（比如用 rsync 命令自动同步）。  
- **限制上传带宽**：在手机客户端“设置-网络”里调上传速度（比如设 1MB/s），避免备份时占满家里网速，影响其他人用网。  
- **开启多设备相册同步**：在 Web 界面“设置-同步”里勾上“多设备相册合并”，不同手机拍的照片会按时间排序，找起来更方便。  
- **隔段时间更新版本**：Immich 迭代快，新功能和安全修复多。更新时只需在服务器上执行 `docker-compose pull && docker-compose up -d`，几分钟就能完成，不用重新配置。  


通过这套方案，手机里的照片视频既能安全备份，又能随时在不同设备上查看，还不用给云服务交钱——适合想自己掌控数据的人尝试。
