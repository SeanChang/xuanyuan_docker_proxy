# XXL-JOB 分布式任务调度框架


## 项目概述  
XXL-JOB 是一款分布式任务调度框架，核心设计目标聚焦于**开发快速、学习简单、轻量级部署及灵活扩展**。作为开源项目，它已稳定运行于多家企业的线上生产环境，真正实现“开箱即用”，无需复杂配置即可快速接入业务系统。


## 核心特点  
XXL-JOB 凭借以下特性成为企业级任务调度的优选方案：  

### 1. 易用性设计  
- **低门槛上手**：支持通过 Web 页面完成任务的 CRUD 操作，界面直观，新手可快速掌握；  
- **动态任务管理**：任务状态（启动/停止）、运行参数可动态修改，即时生效，无需重启服务；  
- **在线开发与发布**：内置 Web IDE（GLUE），支持在线编写任务逻辑代码（Java/Shell/Python 等），实时编译生效，省去部署流程，且支持历史版本回溯。  

### 2. 高可用架构  
- **调度中心 HA**：调度中心采用集群部署，自研调度组件保障高可用，避免单点故障；  
- **执行器 HA**：任务执行节点支持分布式集群部署，自动实现任务负载均衡，确保任务可靠执行；  
- **故障转移**：当执行器节点故障时，自动切换至健康节点继续执行任务，提升系统容错能力。  

### 3. 灵活调度能力  
- **多触发方式**：支持 Cron 表达式调度、任务依赖触发（父任务成功后自动执行子任务）、事件触发（通过 API 手动触发单次执行）；  
- **丰富路由策略**：执行器集群部署时，提供轮询、随机、一致性 HASH、故障转移等 9 种路由策略，适配不同业务场景；  
- **弹性扩容缩容**：执行器节点动态上下线后，调度系统自动感知并重新分配任务，无需人工干预。  

### 4. 全面监控与运维  
- **实时状态监控**：支持任务进度追踪、运行日志实时查看（Rolling 日志），问题定位便捷；  
- **运行报表分析**：提供任务数量、调度次数、执行成功率等数据统计，及调度日期分布、成功/失败趋势图表，辅助运维决策；  
- **告警机制**：任务失败时默认通过邮件告警，预留扩展接口支持短信、钉钉等方式，及时响应异常。  

### 5. 扩展性与兼容性  
- **跨语言支持**：调度中心与执行器通过 RESTful API 通信，支持 Java、Python、Go 等多语言接入；  
- **容器化部署**：提供官方 Docker 镜像，支持 Kubernetes 等容器编排平台，简化部署流程；  
- **线程池隔离**：调度线程池按任务类型隔离，慢任务自动降级至“Slow”线程池，避免堵塞核心调度流程。

## 实际应用与认可  
自 2015 年开源以来，XXL-JOB 已接入超 270 家企业的线上业务，覆盖电商、金融、教育、物流等多个领域，典型用户包括：  
- **互联网企业**：美团点评、京东、滴滴出行、哈啰出行、途虎养车、作业帮、猿辅导  
- **传统行业**：海尔、海信、米其林、中国平安、国家电网  
- **科技公司**：科大讯飞、360 金融、用友、拉卡拉  

这些实践验证了其在高并发、大数据量场景下的稳定性与可靠性。
